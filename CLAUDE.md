# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev        # Start Vite dev server on port 8082
npm run build      # Production build
npm run preview    # Preview production build locally
npm run typecheck  # TypeScript type checking (no emit)
npm run deploy     # Build + deploy to Cloudflare Workers
```

## Architecture

This is the marketing and documentation website for **Labonair** — a self-hosted daemon for managing coding agents (Claude Code, Codex, OpenCode). It is deployed to Cloudflare Workers at `labonair.sh`.

**Stack**: TanStack React Start (SSR) + Vite + React 19 + TypeScript + Tailwind CSS 4 + Framer Motion

**Routing**: File-based routing via TanStack Router. Routes live in `src/routes/`. The `routeTree.gen.ts` file is auto-generated — never edit it manually; it regenerates on `npm run dev`.

**Key source structure**:
- `src/routes/` — All pages. `__root.tsx` is the root layout that server-fetches GitHub release data and star counts for use across the app.
- `src/components/` — Shared UI components. `landing-page.tsx` is the main hero/feature section.
- `src/posts/` — Blog posts as Markdown files with YAML frontmatter. Loaded via `src/posts.ts` using Vite's `import.meta.glob()`.
- `src/releases.ts` / `src/stars.ts` — Server-side GitHub API fetching.
- `src/meta.ts` — SEO metadata helper used by route loaders.
- `src/downloads.tsx` — Platform-specific download link logic.

**Path alias**: `~/*` maps to `./src/*`.

**Deployment**: `wrangler.toml` configures the Cloudflare Workers deployment. Entry point is `@tanstack/react-start/server-entry`. Custom domains: `labonair.sh` and `www.labonair.sh`.

**Sitemap**: Auto-generated by the Vite plugin from a hardcoded route list in `vite.config.ts`.
